## 🚨 用户最受影响的问题 TOP 5

### 1️⃣ **移动端布局完全破损** (致命级)
```
现象: 375px 屏幕上按钮重叠/图表被压扁/无法点击
影响范围: 60% 的用户 (移动端比例)
用户感受: "这个应用根本没法在手机上用"
修复工时: 4h (P0 快速修复)
```

### 2️⃣ **策略计算卡 UI** (关键级)
```
现象: 切换 1m/5m 时间帧时，图表冻结 2-5 秒
影响范围: 100% 高频交易用户
用户感受: "应用不响应，以为崩溃了"
修复工时: 8h (Worker 迁移)
```

### 3️⃣ **新闻全是假数据** (信任级)
```
现象: "2026 年 BTC 达到 5 万美元" (显然是随机生成)
影响范围: 100% 用户
用户感受: "我不信这个应用的任何信息"
修复工时: 4h (接入 CoinGecko News API)
```

### 4️⃣ **预警无声音/通知** (功能级)
```
现象: 预警触发时无任何反馈，用户错过交易信号
影响范围: 预警功能依赖用户 (30%)
用户感受: "预警系统形同虚设"
修复工时: 4h (Notification API + Web Audio)
```

### 5️⃣ **移动端无法交互** (体验级)
```
现象: 手机上无法放大/缩小图表，无法左右滑动查看历史数据
影响范围: 60% 移动端用户
用户感受: "只能看，不能操作"
修复工时: 8h (pinch-zoom + pan 手势)
```

## 🎯 关键决策点

### 问题 1: 是否暂停发布，直到移动端完善？

**决定**: ✅ **是** - 建议延迟 2-3 周

**原因**:
- 目前移动端 35/100 分，无法支撑生产工作流
- 用户会因为"在手机上无法使用"而放弃
- 一旦用户离开，再吸引回来成本高 10 倍

**代价**: 晚 2 周上线

**收益**: 
- 新用户转化率提升 40%+
- 用户留存率提升 30%+

---

### 问题 2: 是用 Mock 数据还是真实数据发布？

**决定**: 🟠 **混合方案** - 核心真实 + 边缘 Mock

**方案**:
```
发布时数据质量:
├─ K线数据 (100% 真实) - CryptoCompare API
├─ 实时价格 (100% 真实) - WebSocket
├─ 宏观因子 (80% 真实) - FRED API + Mock 兜底
├─ 新闻 (70% 真实) - CoinGecko News API + Mock 兜底
└─ 预测市场 (60% 真实) - Polymarket API + 静态数据

=> 平均真实度 82% (可接受)
```

**时间**: 添加这些 API 接入 = 1 周

---

### 问题 3: 本地部署还是云部署？

**决定**: ☁️ **云部署** (Vercel/Railway)

**理由**:
- 本地部署难以保证 99.9% 可用性
- 移动端用户需要 HTTPS (WebSocket/Notification API)
- 自动化 CI/CD 降低风险

**推荐**: 
- 前端: Vercel (免费, 自动 CDN)
- 后端: Railway ($5/月, 自动伸缩)
- 数据库: SQLite + AWS S3 备份
