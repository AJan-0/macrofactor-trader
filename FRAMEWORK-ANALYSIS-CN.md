## 四、移动端特定问题分析

### 4.1 布局问题

| 问题 | 表现 | 影响 | 解决方案 |
|-----|------|------|--------|
| **屏幕宽度不足** | 图表工具栏横排拥挤 | 按钮不可点击 | 改为竖排/折叠菜单 |
| **高度限制** | 图表被压扁 | 看不清蜡烛线 | 全屏图表 + 侧滑面板 |
| **Viewport 配置** | 设备像素比混乱 | 字体模糊 | 添加 `<meta name="viewport">` |
| **虚拟键盘** | 弹起时页面缩放 | 输入框被挡住 | CSS `position: fixed` + `vh` 单位 |

### 4.2 交互问题

| 问题 | 表现 | 影响 | 解决方案 |
|-----|------|------|--------|
| **无触摸反馈** | 用户不知道按钮是否被点中 | 误操作 | 添加 `:active` + `@media (hover: none)` |
| **长按无反应** | 无法复制K线数据 | 不便 | 添加长按菜单 (Context API) |
| **缩放手势缺失** | 无法放大图表 | 看不清细节 | Pinch-zoom + Pan 实现 |
| **滑动误触发** | 页面滑动时触发图表交互 | 困惑 | 添加事件去抖和滑动判定 |

### 4.3 性能问题

| 问题 | 表现 | 影响 | 解决方案 |
|-----|------|------|--------|
| **Bundle 过大** | 首屏白屏 > 5s | 用户放弃 | Code-splitting + 懒加载 |
| **内存溢出** | 应用无响应/刷新 | 崩溃 | 组件卸载时清理 canvas + 事件监听器 |
| **网络抖动** | 图表切割中断 | 看到半加载数据 | 添加中止机制 (AbortController) |
| **电池消耗** | 后台 WebSocket 持续运行 | 快速放电 | 后台标签页限流 + 自动断连 |

### 7.2 错误处理流程

```
API 请求
    ↓
┌─ 重试策略 (3 次) ─┐
│                   │
│  1️⃣ 立即重试      │
│  2️⃣ 250ms 延迟    │
│  3️⃣ 500ms 延迟    │
│                   │
└───────────────────┘
       失败
        ↓
┌─ 降级策略 ─────────┐
│                    │
│ 1️⃣ 返回过期缓存    │
│    (stale-while-   │
│     revalidate)    │
│                    │
│ 2️⃣ 返回默认值      │
│                    │
│ 3️⃣ 报错提示用户    │
│                    │
```

## 十一、关键决策点

### 1. 前端框架版本
- ✅ React 19 + TypeScript (已使用)
- 建议: 启用 React Compiler (5.0 beta) 提升性能
- 风险: 与 DevTools 不兼容, 暂不启用

### 2. 国际化策略
- 当前: 中英文双语 (i18n context)
- 建议: 后端新增数据时自动翻译 (OpenAI API)
- 成本: API 调用费用低

### 3. 移动端优先 vs 桌面优先
- 建议: **移动端优先** (M-to-D approach)
  - 市场数据表明 60% 用户来自移动端
  - 约束迫使更好的 UX
  - 桌面自动适配更容易

### 4. 实时数据可靠性
- 目标: 99.9% 送达率 (4 个 9)
- 方案: WebSocket + HTTP 降级 + 本地队列 + 重试
- 成本: 后端需增加预告缓存 + 事件日志

### 5. 分析平台选择
- 建议: **Plausible** (隐私友好, GDPR 合规)
- 成本: ~$20/月
- 替代: Umami (开源, 自部署)
